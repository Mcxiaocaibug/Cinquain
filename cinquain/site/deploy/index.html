Deploy guide

Bring up Cinquain on your domain without assembling the network by hand.

This page is part of the product surface. It is meant to replace “go read scattered docs” with one direct operator path.

Step 1

Prepare the environment file

cd cinquain
cp .env.example .env

Fill these values before first boot:

  • CINQUAIN_SERVER_NAME
  • CINQUAIN_ACME_EMAIL
  • CINQUAIN_SUPPORT_EMAIL

./install.sh will generate CINQUAIN_BOOTSTRAP_SECRET automatically if it is empty.

Step 2

Run the one-command installer

./install.sh

The curated stack starts Caddy, the homeserver, persistent volumes, TLS handling, and Matrix discovery with one command.

Step 3

Create the first administrator in the runtime

https://your-domain/bootstrap

Use the bootstrap secret stored in .env to unlock the guided setup page, then choose the username and password for the first administrator account.

Step 4

Verify the runtime

./doctor.sh

Then visit these routes:

  • /bootstrap
  • /support
  • /_continuwuity/
  • /.well-known/matrix/client
  • /.well-known/matrix/server

Operations

Backup, upgrade, publish

./backup.sh
./upgrade.sh
./release-image.sh ghcr.io/your-org/cinquain:0.1.0
